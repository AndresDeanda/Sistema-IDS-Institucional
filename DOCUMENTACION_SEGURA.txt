# 1. ARQUITECTURA GENERAL DEL SISTEMA

El Sistema IDS Institucional está desarrollado en Python y funciona como una herramienta de monitoreo de red. Su objetivo principal es capturar tráfico, validar dispositivos autorizados, detectar actividad sospechosa y generar alertas al administrador cuando se presenta un evento relevante.

El sistema se encuentra organizado por módulos, lo que permite separar responsabilidades y facilitar su mantenimiento. El archivo principal `ids_main.py` funciona como punto de entrada del programa y se encarga de inicializar los demás componentes del IDS.

Componentes principales:

* `ids_main.py`: archivo principal que inicia el sistema.
* `config_loader.py`: carga las variables de configuración desde el archivo `.env`.
* `packet_capture.py`: captura paquetes de red mediante Scapy.
* `whitelist.py`: valida IPs y MACs autorizadas.
* `traffic_monitor.py`: registra tráfico y consultas DNS en bitácoras.
* `threat_intel.py`: compara conexiones con IPs peligrosas.
* `forensics.py`: realiza consultas forenses mediante Abuse/Whois.
* `email_alerter.py`: envía alertas por correo electrónico.
* `logger.py`: genera registros del sistema.

La arquitectura del sistema sigue una lógica modular, donde cada archivo cumple una función específica. Esto evita concentrar toda la lógica en un solo archivo y facilita identificar errores, modificar configuraciones o ampliar funcionalidades.

# 2. MODELO OSI APLICADO AL IDS

El IDS trabaja principalmente con información de diferentes capas del modelo OSI:

| Capa OSI                 | Aplicación dentro del IDS                                     |
| ------------------------ | ------------------------------------------------------------- |
| Capa 2 - Enlace de datos | Validación de direcciones MAC mediante lista blanca.          |
| Capa 3 - Red             | Validación de direcciones IP internas y externas.             |
| Capa 4 - Transporte      | Identificación de protocolos como TCP y UDP.                  |
| Capa 7 - Aplicación      | Registro de dominios consultados mediante tráfico DNS o HTTP. |

El módulo de lista blanca se relaciona principalmente con las capas 2 y 3, ya que analiza direcciones MAC e IP. El módulo de monitoreo de sitios trabaja con información de capa 7 al registrar dominios consultados. El módulo de IPs peligrosas analiza conexiones hacia direcciones externas que pueden representar un riesgo de seguridad.

# 5. PROTECCIÓN DE CREDENCIALES

El sistema no debe contener contraseñas escritas directamente dentro del código fuente. Para proteger las credenciales, se utiliza un archivo `.env`, el cual no debe subirse a repositorios públicos.

El archivo `ids_main.py` no contiene contraseñas hardcoded. En su lugar, obtiene los valores de configuración mediante la función `cargar_configuracion()`.

Ejemplo de variables requeridas:

```env
SMTP_HOST=smtp.gmail.com
SMTP_PORT=587
SMTP_USER=correo_remitente@gmail.com
SMTP_PASSWORD=contraseña_de_aplicacion
ADMIN_EMAIL=administrador@dominio.com
NETWORK_INTERFACE=Wi-Fi
WHITELIST_FILE=config/whitelist.csv
BLACKLIST_FILE=config/blacklist.txt
LOG_DIR=logs
REPORT_DIR=reports
DEBUG_MODE=true
```

El archivo `.env` debe permanecer únicamente en el equipo donde se ejecuta el IDS. En GitHub solo debe subirse una plantilla como `env.example`, sin correos reales, contraseñas o claves privadas.

Medidas aplicadas:

* Las credenciales SMTP se cargan desde variables de entorno.
* El correo del administrador puede modificarse sin cambiar el código fuente.
* Las rutas de lista blanca, lista negra, logs y reportes se configuran desde `.env`.
* El archivo `.env` no debe publicarse en GitHub.
* Para Gmail se recomienda usar contraseña de aplicación y no la contraseña principal de la cuenta.

## VALIDACIÓN DE PRIVILEGIOS

El archivo principal valida si el programa tiene permisos suficientes para capturar tráfico de red.

En Windows verifica privilegios de administrador:

```python
if os.name == "nt":
    import ctypes
    if not ctypes.windll.shell32.IsUserAnAdmin():
        print("[ERROR] Este programa requiere privilegios de Administrador.")
        sys.exit(1)
```

En Linux verifica si se ejecuta como root:

```python
else:
    if os.geteuid() != 0:
        print("[ERROR] Este programa requiere ejecutarse como root.")
        print("        Usa: sudo python3 ids_main.py")
        sys.exit(1)
```

Esto es necesario porque la captura de paquetes requiere permisos elevados.

## LIMITACIONES

* Si el archivo `.env` no está configurado correctamente, el sistema no puede iniciar todos sus módulos.
* Si `SMTP_PASSWORD` está vacío, el módulo de alertas no podrá enviar correos.
* Si la interfaz de red está mal escrita, el módulo de captura puede fallar.
* Si no se ejecuta como administrador o root, el sistema no podrá capturar paquetes.
